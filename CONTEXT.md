# DPD Stock System Context

## Overview
โปรเจกต์นี้คือระบบจัดการคลังพัสดุสำหรับ **กองกรรมวิธีข้อมูล (DPD)** เป็นระบบ Web Application ที่ทำงานบนเบราว์เซอร์ ใช้สำหรับจัดการสต็อกสินค้า การเบิก-รับพัสดุ จัดการรายชื่อบุคลากร และการออกรายงาน

## Architecture
- **Serverless / Static Site**: ระบบถูกออกแบบมาให้รันผ่าน GitHub Pages หรือ Web Server ธรรมดา โดยไม่มี Backend API (Node.js, PHP, Python)
- **Database**: ข้อมูลทั้งหมดถูกเก็บไว้ใน `localStorage` ของเบราว์เซอร์เพื่อให้โหลดได้ทันที และซิงค์ข้อมูลเบื้องหลัง (Background Sync) เข้ากับ **Google Sheets** เพื่อเป็นฐานข้อมูลกลาง

## Project Structure
- `index.html`: หน้าหลักหน้าเดียว (Single Page Application) ซึ่งจัดการเมนูต่างๆ ผ่าน JavaScript
- `js/app.js`: ไฟล์ลอจิกหลักทั้งหมด (การคำนวณ, การดึงข้อมูล, การแปลภาษา i18n)
- `css/style.css`: ไฟล์สไตล์หลักของแอปพลิเคชัน
- `assets/`: โฟลเดอร์สำหรับเก็บไฟล์รูปภาพ โลโก้ และวิดีโอพื้นหลัง
- `fix.js`: สคริปต์เสริมสำหรับจัดการส่วนเพิ่มเติม

## Key Functions
- **ภาพรวม (Dashboard)**: สรุปยอดและแจ้งเตือนสินค้าใกล้หมด
- **สต็อกสินค้า (Stock)**: จัดการสินค้า, รับเข้า, เบิกออก
- **ประวัติ (History)**: บันทึกความเคลื่อนไหวทั้งหมด
- **บุคลากร (Personnel)**: จัดการรายชื่อผู้มีสิทธิ์เบิกพัสดุ
